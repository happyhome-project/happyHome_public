# 首次访问弹框直接登录设计

## 目标

首次访问公开示例社区且尚未登录时，用户点击首页欢迎弹框中的“微信一键登录”，应在当前首页直接开始登录流程，不再切换到“我的”页面。

同时将弹框的次操作文案从“免费创建我的社群”调整为“创建我自己的社群”。

## 交互

- 微信小程序支持 `button.open-type.chooseAvatar` 时，主操作直接打开微信头像选择；选择头像后在首页确认昵称并提交登录。
- 用户取消头像选择或昵称确认时，关闭相应流程并留在首页。
- 登录失败时保留首页上下文并展示错误，允许重试，不跳转“我的”。
- H5 等不支持微信头像选择的运行时，主操作在首页展示现有 Web 登录所需的账号、密码和昵称表单，不跳转“我的”。
- 次操作继续进入现有的创建/发现社群流程，只调整显示文案为“创建我自己的社群”。

## 实现边界

- 复用现有用户 store 的小程序 `login` 与 H5 `webLogin` 能力，不改变后端认证协议。
- 将登录表单行为提取为可在首页弹框和“我的”页复用的边界，避免维护两套认证规则；页面外观仅做适配当前弹框所需的最小调整。
- 不修改弹框显示条件、已读版本逻辑、创建社群路由或其他登录入口。

## 验证

- 单元或静态契约测试证明主操作不再调用 `switchTab('/pages/profile/index')`。
- 测试微信头像选择、昵称确认、取消和失败留在首页的行为。
- 测试 H5 主操作在首页打开登录表单。
- 测试次操作文案为“创建我自己的社群”，且既有路由行为不变。
- 运行小程序相关单元测试、类型检查及目标构建。
