# 首页渐变色彩连续性修复设计

> **Historical / point-in-time:** 本设计记录 2026-07-15 首页渐变发布后视觉复核所得的修复方案，仅用于追溯。
> **Current authority:** 以[文档权威索引](../../README.md)、当前首页代码和发布 UI 验证为准。

## 问题与根因

PR #78 已将 `.home-shell` 的背景盒延伸到搜索区下沿，发布版本 `1.0.2607151201` 的上传包也包含相应 WXSS。真实微信运行时仍显得断层，是因为背景盒虽然覆盖了搜索区，原渐变在视觉上过早失去青绿色：

```scss
linear-gradient(170deg, #caeee7 0%, #f1f3ee 58%, var(--hh-color-page) 100%)
```

搜索框位于渐变后半段，实际看到的是接近页面底色的中性色。此前回归测试只验证背景盒几何覆盖，没有验证色彩仍然延续，因此无法发现这类视觉回归。

## 修复方案

保留现有渐变方向、顶部主色、径向高光、背景盒尺寸、DOM 和 sticky 结构，仅调整线性渐变颜色节点：

```scss
linear-gradient(
  170deg,
  #caeee7 0%,
  #dcefe8 58%,
  #edf4ed 84%,
  var(--hh-color-page) 100%
)
```

- `#caeee7`：保留社群标题区域的既有青绿色。
- `#dcefe8 58%`：让格言区域仍明显属于同一青绿色表面。
- `#edf4ed 84%`：搜索框外围保留浅青绿色，但不抢白色搜索胶囊的层级。
- 页面底色只在最后 16% 完成过渡，避免搜索区与下方内容硬切。

这些颜色来自用户参考图的背景取样并约束在现有冷绿色色系内，不新增独立背景条或阴影。

## 保持不变

- `.home-shell` 的 `138rpx` 底部延伸与负外边距保持不变。
- 搜索 sticky 外层继续透明、无阴影；白色只属于搜索输入胶囊。
- 搜索与 Tags 两阶段吸顶、活动区和内容流布局保持不变。
- 不修改云函数、后台、数据库、构建版本或发布逻辑。

## 验收

- 静态契约锁定四个渐变节点，防止中段再次退化为近灰色。
- H5 smoke 同时验证渐变颜色节点、搜索外层透明和背景盒覆盖几何。
- 390px 移动视口视觉复核显示标题、格言和搜索外围属于同一连续青绿色表面，搜索胶囊仍为白色。
- 小程序与 H5 构建、类型检查、完整单测、发布策略和文档检查通过。
