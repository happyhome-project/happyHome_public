# 动态文字排版 H5 样板设计

## 目标

基于最新版 HappyHome 正式文字发布页，制作一套完全本地、无云端写入的交互 H5 样板，用于确认“原文编辑 → 排版生成 → 多页预览 → 发布后瀑布流与详情”的视觉和操作体验。

本样板不复用旧 `prototype/text-note-h5`，不修改 Figma，不修改生产 API，也不加入图片上传、Markdown 或“AI帮你写”。

## 核心原则

- 保留用户原文，不自动改写标题、正文或摘要。
- 排版结果由内容长度、段落和主题共同决定，不是固定背景叠字。
- 本轮 H5 样板将成组文字卡统一调整为 `3:5`，与用户提供的小红书预览参考保持同一长图节奏；本轮不改生产程序现有比例。
- 字号达到可读下限后增加页面，不继续缩小。
- 六种现有主题继续使用正式程序的 SVG 资产，但每种主题拥有不同的封面与正文排版语法。
- 生成过程是真实状态：分析段落、分页、套用主题；样板用可见节奏演示状态，正式实现不强制增加等待。

## 用户流程

### 1. 编辑

- 页面沿用正式 H5 的白底、绿色主按钮、顶部标题和编辑卡。
- 默认填入一篇包含称呼、时间、地点、列表式信息和多个段落的社区停水通知，用户可以自由修改。
- 主操作为“生成排版”，次操作为“存草稿”。
- 标题或正文为空时不进入下一步。

### 2. 生成

- 显示 `3:5` 卡片骨架和三阶段状态：
  1. 正在识别段落结构
  2. 正在为正文分页
  3. 正在套用社区主题
- 完成后进入多页预览，显示实际生成页数。

### 3. 多页预览

- 主预览可左右滑动，并显示 `当前页/总页数`。
- 不再显示重复的页面缩略图条；下一页在主卡右侧露出一小段，提示用户可横向滑动。
- 顶部使用“预览”，右侧提供“编辑”；主题缩略图直接紧跟主卡，不显示“选择排版风格”标题。
- 六种主题均用当前真实标题和正文重新生成整组页面。
- 切换主题时显示短暂“重新排版”状态，页数可随主题变化。
- 话题与地点保留在第二步，UI 与正式图文发布工具一致。
- 底部左侧显示“选择喜欢的排版”，右侧保留 HappyHome 的绿色“发布”。

### 4. 发布后

- 发布返回模拟首页，新帖子插入双列瀑布流顶部。
- 瀑布流只展示生成结果的第一页。
- 点击卡片进入详情；详情使用同一组 `3:5` 文字卡左右翻页，不重复显示裸正文。
- 话题和地点显示在卡片组下方。

## 动态排版规则

- 正文按空行拆分为段落，保留手动换行。
- 封面不把“各位邻居：”这类短称呼作为唯一正文摘要；优先使用第一个信息完整的段落。
- 分页按字符视觉权重和主题容量计算，中文、Emoji、英文和长 URL 权重不同。
- 分页不得丢字、重复或改变正文顺序。
- 过长段落允许按句号、分号等边界拆分；没有自然边界时才按字符拆分。
- 小标题和其后首段尽量同页；孤行不足时整体移到下一页。
- 短内容可以只有封面；中长内容生成封面加若干正文页。

## 六种主题

- `paper` 社区便签：纸张、手写感标题、舒展正文。
- `mint` 邻里日常：清新绿色、正文置于轻透内容区。
- `slate` 今日记录：深色现代排版、紧凑但高对比。
- `headline` 社区小报：居中报头、规则线与新闻式正文。
- `quote` 一句话：文学感封面、正文引用与留白更明显。
- `notice` 通知公告：正式标题、信息分隔清晰、适合时间地点和注意事项。

## 兼容与失败状态

- 生成失败时保留全部输入，提供“重新生成”和“返回修改”。
- H5 刷新后恢复默认演示内容，不保存真实业务数据。
- 桌面浏览器固定展示居中的 `390×844` 手机画布；移动端使用实际可用宽高。
- 支持 `prefers-reduced-motion`。

## 验收

- 编辑、生成中、预览、主题切换、话题、地点、发布后首页、详情八个状态均可操作。
- 使用短句、长通知、连续英文、长 URL、Emoji 和手动换行验证无溢出。
- 主题切换不仅改变背景，还改变标题位置、正文结构、颜色、间距和装饰。
- `390×844` 与桌面内置浏览器均无文字堆叠、横向溢出、按钮遮挡和滚动陷阱。
- 主卡在 `390×844` 下为 `338×563`，左边距 `24`，下一页露出约 `16`；模板条与底部动作按用户提供的参考图校准。
- 浏览器控制台无相关错误或警告。
