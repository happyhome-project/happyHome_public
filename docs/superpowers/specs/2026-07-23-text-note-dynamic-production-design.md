# 动态文字卡正式实现设计

> **Point-in-time:** 本文记录 2026-07-23 已批准 H5 样板向正式小程序落地时的产品与技术边界。
> **Current authority:** 以[文档权威索引](../../README.md)、当前代码、测试和正式发布门禁为准。

## 目标

把纯文字发布从“固定背景叠一段摘要”升级为“原文编辑 → 排版生成 → 多页预览 → 发布确认”的完整体验。长文必须被稳定地放进一组固定 `4:5` 文字卡中，详情页继续使用同一组卡片，不再把正文放到背景外或使用无限增高的文档面板。

## 第一性原则

- 用户的原文是唯一事实源。排版只能分段、分页和套用主题，不能改写、遗漏、重复或改变顺序。
- 可读字号和安全内边距是硬约束。空间不足时增加页面，不继续缩小文字，也不允许溢出背景。
- 生成结果是正文的派生视图，不是新的业务数据。帖子继续保存标题、正文和主题，页面在客户端由相同规则确定性生成。
- 视觉确认与真实发布分离。第二步用于检查排版、主题、话题和地点；“发布”只出现在最后的确认层。
- 历史帖子必须可读。现有 `rich_note` 内容继续兼容；编辑界面保持无工具栏的纯文本体验，不向用户暴露 Markdown。

## 用户流程

1. 第一步只编辑标题和正文，正文支持手动换行，不显示图片入口、Markdown 工具、话题、地点或“AI帮你写”。
2. 点击“下一步”后显示真实排版状态：识别段落、分页、套用主题。计算完成后进入预览。
3. 预览顶部显示一组可横向滑动的 `4:5` 卡片；右侧露出下一页，并显示当前页数。
4. 主题缩略图紧跟主卡，不显示重复页面缩略图，也不显示“选择排版风格”标题。切换主题会重新生成整组页面。
5. 话题和地点沿用图文发布工具 UI。
6. 预览页主按钮为“下一步”。点击后弹出确认层，真正的“发布/保存”只在确认层内执行。
7. 首页、板块页和个人作品仍只展示第一页；详情页展示整组卡片，不重复渲染裸正文。

## 动态分页

- 正文先统一换行符，保留段落和手动换行。
- 封面摘要优先跳过“各位邻居：”一类短称呼，取第一个信息完整的段落，并在安全长度截断。
- 分页按主题容量和字符视觉权重计算。中文、ASCII、空白、换行、Emoji 与长 URL 使用不同权重。
- 优先在段落、句号、问号、叹号、分号等自然边界断页；没有自然边界时才按安全字符边界切分。
- 短正文可以只有一页；长正文生成封面页和若干正文页。
- 所有页面固定 `4:5`，正文页使用主题自身的排版语法和可读字号下限。
- 自动化测试必须证明把所有正文页重新拼接后与规范化原文完全一致。

## 数据与兼容

- 不保存生成图片、不上传分页结果、不新增数据库字段、不修改生产 API。
- `presentation.textNoteTheme` 继续保存所选主题，缺失时回退 `paper`。
- 正文继续走现有 `rich_note` 存储契约，以避免历史数据迁移和云端兼容风险；正式 UI 只提供无工具栏文本编辑面。
- 读取时同时兼容结构化 `rich_note` 与历史纯字符串。

## 失败与边界

- 排版生成失败时保留全部输入并回到可重试状态。
- 连续英文、长 URL、Emoji、超长无标点段落和手动换行不得越界。
- 切换主题后页数可以变化，但原文必须保持一致。
- 本功能 worktree 不执行生产部署、小程序上传、环境/索引变更或数据迁移。真实闭环验证允许创建可精确定位的临时帖子，但必须在同一验证中确认删除。
