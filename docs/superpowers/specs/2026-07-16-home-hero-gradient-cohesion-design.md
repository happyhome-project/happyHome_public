# 首页标题与格言共享色场设计

> **Historical / point-in-time:** 本设计记录 2026-07-16 用户对首页标题、格言和吸顶区域整体配色关系的视觉反馈，仅用于交付追溯。
> **Current authority:** 以[文档权威索引](../../README.md)、当前首页代码和自动化测试为准。

## 问题

当前标题栏使用独立的近似纯色背景，首页主体从另一套渐变重新开始。两者虽然都属于浅青色，但在标题栏下沿发生非单调跳色，因此标题、格言和搜索区看起来像相邻色块，而不是一个连续背景。

参考图的纵向采样从顶部到格言区域保持接近恒定的绿色通道，红色与蓝色缓慢增加，形成连续变亮的青绿色阶。实现应遵循这一关系，而不是只修改格言颜色。

## 色场

- 标题顶部：`#d1f5f2`。
- 标题下沿：`#d9f5f4`。
- 格言中段：`#def4f4`。
- 搜索与标签固定背景：继续使用 `#e6f4f6`。
- 右上薄荷高光保留，但降至 `rgba(116, 220, 185, 0.22)`，并使用固定 `300rpx × 240rpx` 椭圆，使标题栏和主体中的高光坐标一致。

标题栏纵向渐变从标题顶部到标题下沿。首页主体使用相同顶部色，并在 `150rpx + safe-area` 处精确到达同一标题下沿色，之后才过渡到格言和固定背景。这样标题栏覆盖主体时，交界处仍为同一颜色。

## 边界

- 不修改标题、格言、搜索框或标签文字颜色。
- 不修改搜索胶囊、标签文字、容器高度、间距、DOM、滚动容器、`position: sticky`、z-index 或事件逻辑。
- 不修改活动、帖子流、登录流程或数据请求。

## 验收

- 静态契约确认标题栏与主体共享标题顶部、标题下沿和固定尺寸高光。
- H5 smoke 确认两者计算背景均包含同一组色阶，同时保留搜索先固定、标签后固定、反向滚动释放的全部坐标断言。
- 微信与 H5 构建、完整单测、治理检查和文档检查通过。
