# 首页搜索框与板块标签分级吸顶设计

## 目标

首页采用两段式吸顶。社区名称顶部栏继续固定；页面上滑时，搜索框先停在社区名称下方；继续上滑后，板块标签停在搜索框下方。最终形成“社区名称、搜索框、板块标签、帖子流”的稳定纵向层级。

## 交互状态

1. 初始状态：社区名称顶部栏固定，引用、搜索框、活动区域、板块标签和帖子流按正常文档流展示。
2. 搜索吸顶状态：搜索框上缘到达社区名称顶部栏底部时停止上移，其余首页内容继续滚动。
3. 双吸顶状态：板块标签上缘到达搜索框底部时停止上移；搜索框和板块标签同时保持固定，帖子流从两者下方滚动。
4. 向下滚动时按相反顺序解除吸顶，不改变当前搜索内容、选中的板块或帖子流滚动位置。

## 布局方案

采用原生 CSS `position: sticky`，不增加 `onPageScroll` 状态机。搜索框从当前会限制 sticky 生效范围的 `.home-shell` 中拆出，放入覆盖后续首页内容的普通文档流层级；初始视觉仍与首页渐变头部衔接。

固定层级和偏移：

- `.home-topbar`：最高层，保持现有安全区和动态顶部栏高度。
- 搜索 sticky 容器：`top` 等于顶部栏实际占位高度，层级低于顶部栏、高于板块标签。
- `.section-tabs-sticky-shell`：`top` 等于顶部栏高度加搜索 sticky 区高度，层级低于搜索框。

搜索与标签吸顶表面使用不透明或高不透明背景、轻微阴影，确保下面的帖子文字不会透出。搜索框保持完整输入和按钮宽度；板块标签保持现有横向滚动、选中态和点击行为。

## 跨端约束

- 微信小程序和 H5 共用相同结构与 sticky 规则。
- 顶部安全区继续使用现有动态 CSS 变量/内联样式来源，不硬编码设备状态栏高度。
- 不引入滚动事件监听，避免小程序端 fixed/placeholder 切换造成跳动。
- 首页回到顶部、板块切换后的滚动恢复和下拉刷新行为保持不变。

## 测试与验收

- 静态契约测试要求搜索 sticky 容器存在，并要求标签偏移基于“顶部栏 + 搜索区”。
- 微信 DevTools 真实 UI 测试采集三种布局：初始、搜索框吸顶、搜索框与标签双吸顶。
- 双吸顶时验证：顶部栏位置不变；搜索框紧贴顶部栏下方；标签紧贴搜索框下方；切换板块不改变两个吸顶位置。
- 回归运行小程序类型检查、单元测试、微信构建、H5 构建、首页静态视觉测试及 `git diff --check`。

## 非目标

- 不修改搜索逻辑、搜索结果页或搜索文案。
- 不修改社区切换、活动区域、板块数据或帖子流业务逻辑。
- 不重新设计顶部栏、搜索框或标签的视觉样式。
