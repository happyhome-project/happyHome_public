# 首页标题至搜索框渐变连续性设计

> **Historical / point-in-time:** 本设计记录 2026-07-15 已确认的视觉修复方案，仅用于追溯，不覆盖后续产品或实现决策。
> **Current authority:** 以[文档权威索引](../../README.md)、当前首页代码、发布 UI 检查和测试为准。

## 目标

首页顶部的青绿色渐变需要从社群标题自然延伸，覆盖格言和搜索框所在区域，再过渡到活动与内容区的页面背景。搜索输入胶囊继续保持白色，但其透明 sticky 外层下方不能在格言结束处突然露出灰色页面底。

## 结构约束

搜索 sticky 容器必须继续位于 `.home-shell` 之外。若把搜索框放回渐变容器内部，原生 `position: sticky` 会受父容器底边限制，无法随后续内容继续吸顶。

因此不改变 DOM 层级，也不给 `.home-search-sticky-shell` 添加背景。改为让 `.home-shell` 的背景盒向下延伸搜索表面的完整高度 `138rpx`，同时使用等量负外边距抵消布局占位：

- `.home-shell` 底部 padding：`138rpx`；
- `.home-shell` 底部 margin：`-138rpx`；
- 搜索 sticky 容器仍为下一兄弟节点，位置和 sticky 偏移保持不变；
- 初始状态下，搜索容器底边与 `.home-shell` 的渐变背景底边重合。

`138rpx` 来自当前搜索 sticky 表面高度：上下各 `24rpx` padding，加 `90rpx` 搜索框高度。两处尺寸由静态契约绑定，避免后续漂移。

## 视觉边界

- 保留现有渐变配色和方向，不创造新色板。
- 保留搜索外围透明、无阴影；白色只属于搜索输入胶囊。
- 保留 Tags 外围透明。
- 不修改格言文字、引用线、活动卡片、归档内容或底部导航。

## 验收

- 静态测试证明 `.home-shell` 向下延伸并以相同数值抵消占位，且搜索 sticky 表面仍为 `138rpx`。
- H5 真实浏览器初始状态中，`.home-shell` 底边与搜索 sticky 容器底边重合。
- 搜索外围 computed style 仍透明、无阴影。
- 搜索与 Tags 两阶段吸顶坐标、反向释放行为继续通过。
- 微信小程序与 H5 构建、类型检查、完整单测和发布策略检查通过。

## 非目标

- 不给 sticky 容器增加常驻渐变或白色背景。
- 不增加 `onPageScroll` 状态机。
- 不修改云函数、后台、数据库或生产数据。
