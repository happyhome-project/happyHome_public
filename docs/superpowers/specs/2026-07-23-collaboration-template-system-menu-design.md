# 协作模板移入系统管理设计

## 目标

将超级管理员侧边栏中的“协作模板”从顶层菜单移入可折叠的“系统管理”区域，同时保持现有路由、页面和权限不变。

## 导航行为

- “协作模板”作为“系统管理”的第一个子项，位于“管理员管理”和“样板社群引导”之前。
- 菜单项仍使用 `/collaboration-templates` 和现有 `menu-collaboration-templates` 测试标识。
- “系统管理”仅对超级管理员显示，因此协作模板的超级管理员边界不变。
- 直接打开或刷新协作模板、管理员管理、样板社群引导页面时，系统管理区域自动展开，确保当前菜单项可见。
- 用户仍可在其他页面手动展开或收起系统管理，状态继续写入现有本地存储键。

## 变更边界

只修改 `admin-web/src/views/Layout.vue` 和导航契约测试。协作模板页面、路由定义、API 与服务端权限不变。

## 验证

- 先增加失败的导航契约测试，证明协作模板必须位于系统管理区，并要求系统路由自动展开。
- 运行 admin-web 单元测试、类型检查、生产构建和 `git diff --check`。
