# 社群首字默认头像与分享封面设计

## 背景

当前首页在社群未设置 `coverImage` 时显示名称首字，但微信分享卡片会回退到 `/static/logo.png`。这会让分享封面与社群身份脱节，也可能在大尺寸卡片中暴露不适合放大的默认 Logo。

## 目标

- 已设置社群头像时，首页和分享继续使用该头像。
- 未设置头像时，所有社群头像入口统一显示社群名称的首个有效字符。
- 分享卡片使用与界面头像一致的首字图片，不再使用默认 Logo。
- 社群头像无法解析或加载时，也回退到首字样式。
- 不迁移历史数据，不向云存储写入派生图片。

## 非目标

- 不修改用户头像或帖子封面逻辑。
- 不新增社群头像字段。
- 不为不同社群生成随机配色。
- 不改变已有头像的裁切、上传或存储行为。

## 视觉规范

采用现有 HappyHome 品牌色和系统中文无衬线字体：

- 背景：现有浅品牌绿 `$hh-accent-wash`，画布值固定为 `#E8F8F0`。
- 文字：现有深品牌绿 `$hh-accent-ink`，画布值固定为 `#1F7A50`。
- 字体：`-apple-system, BlinkMacSystemFont, "PingFang SC", "Hiragino Sans GB", "Segoe UI", sans-serif`。
- 字重：600。
- 字符在画布中水平、视觉垂直居中，四周保留充分留白。
- 界面头像保持当前圆形容器。
- 分享封面固定为 `500 × 400` 像素横向 PNG；首字字号 `160px`、字重 600，并依据 Canvas 文本测量结果居中。

配色固定，不根据名称随机变化，以保持品牌一致性和可预期性。

## 字符规则

1. 对社群名称执行首尾空白清理。
2. 取第一个完整 Unicode 字素簇，避免 emoji 或组合字符被截断。优先使用 `Intl.Segmenter`；目标运行时不支持时，使用项目内轻量回退实现，将组合附加符、变体选择符、肤色修饰符和 ZWJ 连接序列归入同一字素。
3. 名称没有有效字符时使用“群”。
4. 中文、英文、数字和 emoji 均使用同一规则。

该规则由独立纯函数提供，供界面回退和分享图片生成共用。

## 运行时设计

### 界面头像

社群有可用 `coverImage` 时渲染图片；没有图片或图片加载失败时渲染首字回退。本次明确覆盖以下社群头像入口：

- 首页顶部社群头像 `pages/index/index.vue`。
- 社群切换列表 `pages/community-switch/index.vue`。
- 加入/发现社群列表 `pages/onboarding/index.vue`。

搜索结果中的文字封面属于帖子或板块封面，不是社群头像，本次不修改。用户头像也不在本次范围内。三个社群头像入口共享相同字符规则和视觉 token，避免各页面自行截取字符串或使用不同样式。

### 分享图片

首页和“我的”页都可发起社群邀请分享，二者共享同一图片准备逻辑。`onShareAppMessage` 必须同步返回，因此 Canvas 工作在页面加载、社群切换或社群信息变化时提前异步完成；分享回调本身只读取已就绪结果。

预生成监听键为“社群 ID、规范化名称、`coverImage` 和视觉版本”。键变化后立即清空旧的已就绪结果，再按以下优先级准备 `imageUrl`：

1. 当前社群存在 `coverImage`，成功解析且实际图片加载探测成功时，使用解析后的头像地址。
2. 否则使用隐藏 Canvas 生成首字分享封面，并通过小程序运行时导出临时 PNG 路径。
3. 如果准备尚未完成或 Canvas 生成失败，省略自定义 `imageUrl`，交由微信使用页面默认截图；不得重新回退到 `/static/logo.png`。

Canvas 使用 uni-app 微信端兼容的 `canvas-id`、`uni.createCanvasContext` 与 `uni.canvasToTempFilePath`。节点在页面中保持挂载并移到可视区域外，不使用 `display: none` 或条件卸载，以免导出空图。生成结果按监听键缓存本地临时路径，避免同一页面生命周期内重复绘制。社群切换或改名后，缓存键变化并生成新图。

头像地址解析后，使用小程序图片信息接口执行实际加载探测。解析成功但加载失败与页面 `<image @error>` 都会把对应监听键标记为失败，并触发首字分享图准备；该状态只影响客户端回退，不修改服务端 `coverImage`。

### 并发与过期结果

图片解析、加载探测和 Canvas 导出均为异步操作。每次准备图片时记录完整监听键；异步完成后再次核对，只有仍与当前社群一致时才更新分享图片，避免快速切换社群造成封面串群。准备期间分享回调不复用上一社群图片，而是省略 `imageUrl`。

## 错误处理

- 云文件解析失败：记录现有客户端告警，并生成首字分享封面。
- Canvas 创建或导出失败：记录不包含敏感数据的告警，并让微信采用默认页面截图。
- 界面图片加载失败：仅该入口切换为首字回退，不删除或修改服务端头像字段。
- 任何回退失败都不阻断分享动作。

## 测试

### 单元测试

- 中文名称 `明士班` 得到 `明`。
- 带首尾空白的名称得到清理后的首字。
- 英文、数字和 emoji 得到第一个完整字素簇。
- 组合附加符、肤色修饰符和 ZWJ emoji 在不支持 `Intl.Segmenter` 的回退路径中仍保持完整。
- 空名称得到 `群`。
- 有头像、无头像、头像解析失败和头像加载探测失败时遵循分享图片优先级。
- 异步结果只写入仍匹配当前社群的状态。

### 页面契约测试

- 首页和“我的”页分享都不再以 `DEFAULT_COMMUNITY_SHARE_IMAGE` 作为无头像回退。
- 首页、社群切换页和加入/发现页使用统一首字函数和统一回退样式。
- `onShareAppMessage` 返回已准备好的头像或首字临时图片；准备中或生成失败时省略 `imageUrl`。

### 运行时验证

- 在小程序开发者工具创建或选择一个无头像临时社群。
- 核对首页、社群切换和加入/发现页的圆形首字头像。
- 分别从首页和“我的”页分享给微信会话，核对卡片标题、首字、字体、配色、留白和目标路径。
- 再验证一个已有头像社群，确认分享仍展示原头像。
- 清理本次验证创建的临时数据。

## 验收标准

- 无头像社群在所有相关界面均显示名称首个完整字符。
- 无头像社群的微信分享卡片展示 A 方案的浅绿底深绿字封面。
- 已有头像社群行为不变。
- 头像解析失败不会展示默认 Logo，也不会阻断分享。
- 自动化测试通过，并完成至少一次真实微信分享卡片闭环验证；若外部环境不可用，必须明确报告未覆盖的运行时验证。
