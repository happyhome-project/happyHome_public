## 0. 一句话产品定位

**美好 Home 是服务社群的助手**：把社群产生的有价值信息沉淀下来（防被微信群刷走），并处理轻度实时协作（拼车、活动报名、签到）。

---

## 1. 目标用户（3 类场景，同一套产品）

| 场景 | 典型板块 | 核心用户 | 使用节奏 |
|------|---------|---------|---------|
| **邻里社群** | 闲置交易、好店推荐、邻里互助、小区公告 | 业主群 | 日常低频，活动时高频 |
| **课程班** | 课件归档、学员作业、课堂签到、答疑 | 老师 + 学员 | 每周上课期间高频 |
| **兴趣小组** | 活动召集、作品分享、路线推荐、打卡 | 群主 + 成员 | 活动期高频，平时低频 |

**关键洞察**：
- **管理员是内容创作主力**（20% 的人贡献 80% 的内容）
- **社群活跃度取决于管理员**（"论坛火不火看管理员"）
- 用户规模小（30-100 人），内容量小（一天 1-10 条）—— 这意味着**首页即精华**

## 2. 调性锚点（Visual Tone）

| 维度 | 决策 |
|------|------|
| 关键词 | **温暖、沉淀、有人味** |
| 视觉锚点 | 小红书语感 + 木质书架气质 |
| 主色 | **木色橙 `#E07A5F`** — 温暖但有重量，契合"沉淀"价值 |
| 做 ✅ | 单列卡片流、圆润、暖色调、友好文案（"还没有动态哦"）|
| 不做 ❌ | 双列瀑布流（丢失元信息）、花哨动画、高饱和红绿灯、冷灰色 |

## 4. 信息架构（核心决策）

### 板块双类型系统

所有板块由 admin 创建，必须二选一：

**实时协作类**（Real-time）
- 举例：拼车、签到、活动报名、临时通知
- 特点：**时效性强，过期作废**
- 首页位置：**顶部置顶脉冲区**（红点脉冲 + 暖橙渐变背景）
- 多个激活时：**全部置顶**，用小 icon 标签区分（🚗 拼车 / ✍️ 签到）

**沉淀展示类**（Evergreen）
- 举例：好店推荐、课件归档、作品集、小区公告
- 特点：**长期有价值，365 天可用**
- 首页位置：**分组卡片**，每组 top 2-3 条 + "查看全部"
- 永远常驻显示

### 实时协作类的三态模型

| 状态 | 首页显示 | 发帖 | Tab | 何时用 |
|------|---------|------|-----|-------|
| **激活** 🔥 | 置顶脉冲区显眼显示 | ✅ | ✅ | 活动当天、功能启用期 |
| **休眠** 🌙 | 折叠到"暂未启用"区 | ✅ | ✅ | 非活动期、待下次使用 |
| **归档** 📦 | 首页不显示 | ❌ | ❌（进"历史板块"）| 彻底结束不再用 |

**切换方式**：admin 手动切换（不搞自动过期）

### 沉淀展示类

无状态切换，永远常驻。只有 admin 删除才会消失。

## 5. 首页布局规则（自适应板块数）

```
[顶栏：社区名 ▾ + 成员数/今日新]
      ↓
[实时置顶区（仅当有激活的实时板块）]
  🔴 正在进行 · 今日活动 
  ├── 🚗 拼车卡（路线图 + 剩余座位 + 价格）
  └── ✍️ 签到卡（进度 8/15 + 位置）
      ↓
[沉淀分组卡 1] 🏷 闲置交易 · 42 条
  ├── 帖子 A（图文 + 价格）
  └── 帖子 B
      ↓
[沉淀分组卡 2] 🏪 好店推荐 · 28 条
  ├── ...
      ↓
[沉淀分组卡 N]
      ↓
[休眠折叠区 🌙（仅当有休眠板块）]
  "其他板块（暂未启用）：拼车、读书会"
      ↓
[底部 Tab: 首页 / 我的]  + FAB 发帖按钮
```

### 自适应规则

| 板块数 | 处理 |
|--------|------|
| **0 板块** | 空态 + "管理员还未创建板块"引导 |
| **1 板块** | 不渲染 tab，banner 标示 + 直接 feed |
| **2-6 板块** | Tab 横滑 + "全部" |
| **7+ 板块** | Tab 横滑 + 右侧渐隐 + "更多"下拉 |

### 绝对不能做的

❌ **硬编码具体板块名**（如"紧急服务"）— 板块由 admin 动态创建
❌ **假设固定板块数**（如"4 个板块网格"）— 不同社群数量不同
❌ **双列瀑布流** — 丢失结构化元信息（价格/时间/路线）

## 6. 卡片设计规范

### 骨架（所有卡片共享）

```css
.card {
  background: #FFFFFF;
  border-radius: 14px;
  padding: 14px 16px;
  box-shadow: 0 1px 4px rgba(0,0,0,0.03);
  border-left: 3px solid var(--板块类型色); /* 可选场景色条 */
}
```

### 场景色条（可选，区分板块类型）

- 橙 `#E07A5F` — 闲置交易、推荐
- 蓝 `#6E8CA0` — 拼车、课件、信息
- 绿 `#52C41A` — 活动、签到、成功
- 黄 `#FAAD14` — 求助、警告

### 三段式结构

```
[header] 标题 + 时间右上角
[body]   描述 / 路线图 / 进度条 / 图片
[footer] 头像 + 作者 + 操作按钮（分隔线之上）
```

### 5 种业务卡片变体

1. **闲置交易**：图文混排 + 橙色价格大字
2. **拼车**：路线图（A ○──→ B ●）+ 芯片（时间/座位/价格）
3. **活动**：三图小图 + 报名进度条（渐变填充）
4. **求助**：黄色左色条 + 回复数标签
5. **课件**：蓝色左色条 + "查看"按钮 + 已上传状态

## 8. 已决定 vs 待探索

### ✅ 已拍板
- 主色 #E07A5F 木色橙
- 调性：温暖、沉淀、有人味
- 板块双类型（实时 / 沉淀）
- 实时三态（激活 / 休眠 / 归档），admin 手动切
- 首页布局：置顶脉冲区 + 沉淀分组卡 + 休眠折叠
- 6 个 Tab / 胶囊自适应规则
- FAB 发帖按钮（可选）

### 🟡 待探索（适合在 Claude.ai/design 里做）
- **详情页布局**：帖子详情该怎么排？评论怎么处理？编辑/删除按钮位置？
- **发帖流程**：选板块 → 填表单的完整路径怎么优化？是否用底部弹出？
- **Admin 后台**：板块状态切换 UI、数据面板、成员审批流程
- **Onboarding**：新人加入社区的引导流程
- **创建板块向导**：admin 怎么配置板块类型 + widget + 状态
- **搜索/筛选**：如何在沉淀板块里找东西
- **社区切换 Drawer**：多个社区的切换面板
- **空态插画**：每种空态配什么插画

## 9. 使用这份简报的建议

**在 Claude.ai/design 里**：
1. 新对话开头，粘贴本文件全部内容
2. 指定要设计的页面（比如"我要设计帖子详情页"）
3. 让 Claude 用 tokens + 规则生成 HTML/React artifact
4. 用 `prototype/realtime-evergreen.html` 的视觉语言作为参考

**约束 Claude.ai 的提示词模板**：
```
我在设计"美好 Home"微信小程序的 [XXX] 页面。
请严格遵守：
- 使用附件里的 design tokens（--primary 木色橙 / rgba 文字色 / 暖色阴影）
- 遵循"温暖、沉淀、有人味"调性
- 不要硬编码具体板块名（板块是 admin 动态创建的）
- 卡片用 border-radius: 14px + shadow 0 1px 4px rgba(0,0,0,0.03)
- 标签用非对称圆角 6px 2px 6px 2px
- 输出单页 HTML artifact，内嵌 style，可直接预览
```
